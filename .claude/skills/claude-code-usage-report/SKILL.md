# claude-code-usage-report

Interactive HTML "Claude Code Usage Report" ‚Äî session cost, token usage, efficiency, usage patterns, and burn highlights ‚Äî read from `~/.agents/.claude-code-usage-report/state/stats.csv`. Self-contained HTML, no external libs, persisted at `~/.agents/.claude-code-usage-report/reports/report-<timestamp>.html` (mode 0600 on POSIX; a no-op on Windows/NTFS). Day/Month toggle. Full column list lives in `stats.mjs`; trailing statusline-fed columns are forward-only (0/blank for older rows), duration/turns/tool_calls backfill from transcripts. Trailing `est_cost_usd` holds a token-estimated cost for sessions with no billed `total_cost_usd` (no statusline); the report shows billed-else-estimated, priors calibrate on billed only.

`SKILL_DIR` = this skill's directory. Runs via `node` (ships with Claude Code, on PATH). Fresh machine: install first ‚Äî `node <SKILL_DIR>/scripts/stats.mjs install` (writes SessionEnd hook + creates dirs; see INSTALL.md).

## Steps

1. Render: `node <SKILL_DIR>/scripts/stats.mjs report`. The command first refreshes stats.csv from all transcripts + any lingering cost-state snapshots (excluding the active session), then renders ‚Äî no manual backfill needed. It also refreshes the OAuth usage snapshot (5h/7d gauges + per-model quotas) best-effort each run when creds are present (`~/.claude/.credentials.json`) ‚Äî a silent no-op when logged out or on network/auth failure (prior snapshot kept); no `USAGE_REPORT_OAUTH=1` needed for the `report` path. Per-model weekly quota rows stay blank on plans where the API returns `seven_day_{sonnet,opus,omelette}` as null. The mid-flight active session is skipped (its transcript/cost-state are incomplete); everything else is brought current. **Active-session detection** (Phase I): a session is active iff its `cost-state/<sid>.json` is present and fresh (< 900s ‚Äî the statusline rewrites it per render, so a recent one = live; this tolerates a mid-session pause that the old 180s transcript-mtime window mislabeled as closed). `record` deletes cost-state at SessionEnd, so a stale lingering cost-state (crash/hook-missed) is folded in, not excluded. Falls back to the 180s transcript-mtime window only when no cost-state exists at all (no statusline wired).
2. Script writes the report, prints its absolute path, and auto-opens it in the OS default browser (`USAGE_REPORT_BROWSER=firefox|chrome|...` to override). If browser launch fails, the path is still printed ‚Äî open manually.
3. Manual `backfill` rebuilds from all transcripts unconditionally ‚Äî escape hatch for after migration or a corrupted stats.csv. Run while no Claude Code session is active.

### Pre-op cost estimate

`node <SKILL_DIR>/scripts/stats.mjs estimate <category>` (planning|execution|verification|orchestration|other; prefix ok) prints historical p50/p90/mean cost + typical turns/output for that op category, derived from recorded `total_cost_usd` redistributed across transcript ops. Priors cache to `~/.agents/.claude-code-usage-report/state/priors.json`, lazy-rebuild when stale (>7d or a newer transcript landed); `stats.mjs priors` forces a rebuild. With a live statusline snapshot it also adds a context re-read floor for the current op. Each category also carries an **out-of-sample calibration** (Phase E): the most recent ~20% of cost-calibrated ops are held out, p50/p90 refit on the rest, and scored for coverage / bias / pinball ‚Äî printed by `priors` (per-category `p90 cov` / `bias` columns) and `estimate` (an `OOS calibration` row). Categories with p90 coverage < 0.70 are flagged (estimate p90 unreliable); p50 coverage well off 0.5 + nonzero bias signals the cost distribution has drifted, so re-backfill before trusting the range. Each category also carries an **empirical-Bayes shrinkage** (Phase H): the raw percentile p50/p90 are location-shifted toward a cross-category grand mean (`FC.fitPrior` between-category variance + normal-normal conjugacy; shift ‚àù per-category sampling uncertainty `withinVar/n`). Stored under `categories[cat].shrink`; `estimate` prints a `shrunk (EB, small-n)` row when `n_cost < 20`. Large-n or low-within-variance categories keep the raw values (data-weight ‚âà 1); noisy small-n categories pull toward the grand mean.

Live capture is automatic via the SessionEnd hook (`stats.mjs record`): reads the full statusline JSON snapshot from `~/.agents/.claude-code-usage-report/state/cost-state/<sid>.json` (written verbatim by your statusline ‚Äî last write per session wins) for cost/duration/lines/rate-limits/context, plus tokens/turns/tool_calls/last_model + transcript facets (tool mix, errors, subagents, skills, project cwd/branch, compactions ‚Üí `facets_json`) from the session transcript JSONL. Also appends raw statusline JSON to `~/.agents/.claude-code-usage-report/state/sessions.jsonl` (archive) and deletes the cost-state file. No manual step for new sessions.

### Rate-limit forecast

`report` also fits an empirical-Bayes rate-limit forecast (claumon MODEL v2.1 port, `scripts/forecast.mjs`) shown in the "Rate-limit forecast at reset" card. Per gauge (5h / 7d) it projects utilization at the reset boundary with an 80% credible interval (monotone Gamma-process MC) + ETA-to-100% and ETA-to-80%. Input is the OAuth usage-snapshots series (`state/usage-snapshots.jsonl`, grouped by normalized `resets_at` into completed windows + the open window); when OAuth polling is off it falls back to a **prior-only** view from the statusline `rl_5h_pct`/`rl_7d_pct` series (rate prior + n windows, no projection ‚Äî no live window to project without `resets_at`). Fit persists to `~/.agents/.claude-code-usage-report/state/forecast.json`, lazy-rebuilt on the same >7d-or-newer-data rule as priors; `stats.mjs forecast [--force]` prints/forces the fit. Empty-state when <2 rl-bearing windows. Needs ‚â•2 completed windows + an open window for a full projection ‚Äî run `fetch-usage --oauth --save` periodically (a few times per window) to accumulate them.

### Burn highlights

The "Burn highlights" section (near the top of the report) names the top reasons your tokens burned in the currently selected timeframe (7d / 30d / all ‚Äî the range picker). It's entirely client-side: `app.js` scores 4 candidate drivers (heavy-session concentration, outlier sessions vs. your all-time median, context compactions, long/high-turn sessions) against the sessions in the selected range, then shows the top 3 as cards, badge (high/med/low) reflecting each driver's severity. Recomputes on every range change, unlike the rest of the report's static sections.

## Capture contract (statusline)

Hard requirement for live cost/duration/lines/rate-limit/context capture: **your statusline must write the raw statusline JSON payload (the stdin it receives) to `~/.agents/.claude-code-usage-report/state/cost-state/<session_id>.json` on each render** ‚Äî last write per session wins (‚âà final snapshot). A cross-platform statusline ships at `<SKILL_DIR>/scripts/statusline.mjs` (model, context-usage bar, cost, rate limits, dir, worktree, git status). `install --with-statusline` copies it into `~/.claude/` and wires the `statusLine` setting to `node "<path>"` (skipped if you already have a statusLine command ‚Äî add the file write to your own statusline instead). Optional: without it, `record` exits early and the report renders from transcripts only (cost/duration/lines/rate-limits blank, populated once a statusline is wired).

## Conventions

- **Token-legend pills** (Token Economics) filter the bar chart with **solo/deselect** semantic, not independent per-pill hide/show: empty `TOK_ACTIVE` = all series visible; click a pill ‚Üí solos that series; click again ‚Üí drops it, back to all when set empties; `All` resets. `tokVis()` returns `Array.from(TOK_ACTIVE)` (or all keys when empty) ‚Äî must be an array, not a Set (`tokenBars` calls `.indexOf`). Do not change to independent-toggle without an explicit ask.
- **Token-yield pills** (Rate-limit section, "Token yield per rate-limit %") reuse the same solo/deselect pill semantic (`TY_ACTIVE`), filtering per-model lines. Metric = **Mtok per 1% of the 5h/7d window**, computed from per-session gauge **deltas** (drop = window reset, delta = current pct), aggregated `Œ£tok/Œ£delta%` ‚Äî NOT `tok/cumulative-pct` (r5/r7 are account-global cumulative gauge readings, so a raw ratio is meaningless). **Claude models only** (`/^claude/`): r5/r7 is the shared Claude account quota; non-Claude models (glm/ollama/etc.) don't draw from it, so their rows are excluded (their gauge reading is just the ambient Claude value). 7d/5h toggle (`showTY`), default 7d. All client-side in `app.js` (`tokenYield`/`svgTokenYield`/`renderTY`); no stats.csv column added.
- **$/hour** ‚Äî no per-session duration cap (legit sessions can exceed 8h); only the `$0`-cost filter drops idle/hung sessions.
- **$/line** gated at 5% line-coverage (`LINE_COV=0.05`); below shows `‚Äî`.
- **Rate-limit panel** (5h/7d) is forward-only from the statusline `rate_limits` field (Claude Code v2.1.80+, Claude.ai Pro/Max only ‚Äî absent for API-key/Bedrock/Vertex and some Max 20x oauth users). Empty-state notice when no rl-bearing sessions; `rl_5h_pct`/`rl_7d_pct` are floats, read with `_fnum` (not `_inum`), displayed `toFixed(1)`. Capped `>=99.5` (float noise), near-cap `>=80`. The **forecast card** beneath it (Phase D) projects each gauge to its reset boundary from OAuth usage-snapshots; projected values can exceed 100% (= demand past the window limit, model v2.1).
- All rendering client-side: report embeds `var SESSIONS=[...]` JSON + a JS IIFE that aggregates and renders every section; date-range controls re-filter ‚Üí re-aggregate ‚Üí re-render.
- **Pricing** (`PRICE` in `stats.mjs`, $/MTok `[input, output, cache_read, cache_create]`): base rates verified against claude.com/pricing#api (Opus 4.8 $5/$25, Sonnet 5/4.6 $3/$15, Haiku 4.5 $1/$5, Fable 5 $10/$50; cache_read=0.1√ó, cache_create-5m=1.25√ó). Resolved layered (Phase F, lowest‚Üíhighest): embedded `PRICE` ‚Üí `state/pricing-cache.json` (if <24h, from `fetch-pricing --oauth`) ‚Üí `PROMOS` (time-bounded intro rates, in-window) ‚Üí manual `state/pricing.json` override. **Sonnet 5 intro pricing ($2/$10) is reflected via `PROMOS` through 2026-08-31** (auto-expires). `stats.mjs fetch-pricing --oauth` (same `USAGE_REPORT_OAUTH=1`/`--oauth` gate as `fetch-usage`) pulls claumon's remote pricing.json, reduces model-ids to the skill's family keys (latest model-id per family), writes the cache; `stats.mjs pricing` prints the resolved table + which layer set each family. `PRICE_ABOVE` is the long-context (>200k) tier mechanism but is **empty** ‚Äî no current Claude model has a >200k premium; kept so a tier can be re-added via `pricing.json` if reintroduced. Estimated cost (`est_cost_usd`, sessions with no billed cost) folds in **subagent-run transcript tokens** (Phase G): each subagent-run transcript under `<project>/<sid>/**/*.jsonl` is summed per assistant message with `_msg_cost_tiered` (the long-context tier applies per request; subagent rows are `isSidechain` relative to the parent, so the sidechain skip is scoped to the main transcript only). Recomputed each backfill for no-billed sessions; billed sessions skip it (total_cost_usd wins, priors calibrate on billed only). cache-create is not split 5m/1h (the breakdown isn't captured). Override rates without editing code via `<state>/pricing.json`: `{ "base": {"opus":[...]}, "above_200k": {"sonnet":[...]}, "long_context_threshold": 200000 }` (bare `{"opus":[...]}` map also accepted as base); only 4-number arrays honored, malformed files ignored.

## Scope & the claude-code-usage-report-suggestions skill

This skill: **collect, visualize, estimate** from local session-cost data (record + backfill stats.csv, render the HTML report, `estimate`/`priors` for pre-op cost prediction ‚Äî estimation derives from the same recorded cost + transcripts, so it lives here with the data layer). Deciding *what new insights to add* and *how to improve the report* belongs to the sibling **`claude-code-usage-report-suggestions`** skill. The report's "Usage roadmap" section is sourced from it: `report` shells out to `claude-code-usage-report-suggestions/scripts/improve.mjs roadmap --json` (resolved next to this skill; no hardcoded install path) and embeds the result (section omitted gracefully if that skill is absent). To add a metric/chart, invoke `/claude-code-usage-report-suggestions`.
